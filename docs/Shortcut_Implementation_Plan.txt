APPLE SHORTCUTS WORKOUT LOGGER - IMPLEMENTATION PLAN
=====================================================

VERDICT: USE SHORTCUTS + SCRIPTABLE (HYBRID APPROACH)
------------------------------------------------------

Pure Apple Shortcuts can handle the menu, prompts, and CSV appending. But
parsing JSON and looping through exercises with dynamic prompts is painful
in Shortcuts alone -- the "Repeat with Each" action combined with "Ask for
Input" works, but building the JSON parsing logic in Shortcuts is brittle.

RECOMMENDED SETUP:
  - Scriptable (free iOS app) handles the logic: reads JSON, builds prompts,
    formats CSV lines
  - Apple Shortcuts calls Scriptable, then handles file I/O and notifications
  - Apple Shortcuts Automation triggers the reminder

This gives you reliable JSON parsing, string formatting, and looping in
JavaScript, while Shortcuts handles what it's good at: file access, iCloud
Drive, notifications, and time-based automations.


========================================================================
FILE LOCATIONS (iCloud Drive)
========================================================================

Programme JSON:  iCloud Drive/Shortcuts/Workout_Program.json
Workout log CSV: iCloud Drive/Shortcuts/Workout_History.csv

IMPORTANT - PATH REALITY:
  On iPhone, Shortcuts can only natively read/write files in:
    iCloud Drive/Shortcuts/

  On Mac, this maps to:
    ~/Library/Mobile Documents/iCloud~is~workflow~my~workflows/Documents/

  Your current files are at ~/Desktop/Fitness/ which is NOT directly
  accessible to iPhone Shortcuts. You have two options:

  OPTION A (recommended): Move the working files into iCloud Drive/Shortcuts/
    - Put Workout_Program.json in iCloud Drive/Shortcuts/
    - Put Workout_History.csv in iCloud Drive/Shortcuts/
    - On Mac, create a symlink so they also appear on your Desktop:
        ln -s ~/Library/Mobile\ Documents/iCloud~is~workflow~my~workflows/Documents/Workout_Program.json ~/Desktop/Fitness/Workout_Program.json
        ln -s ~/Library/Mobile\ Documents/iCloud~is~workflow~my~workflows/Documents/Workout_History.csv ~/Desktop/Fitness/Workout_History.csv

  OPTION B: Use the Shortcuts "File" actions with "Ask Where" disabled and
    specify the full iCloud path. Shortcuts CAN access broader iCloud Drive
    paths using the "Get File" action with a path like:
        /Fitness/Workout_Program.json
    This works if the Fitness folder is in your iCloud Drive root (not Desktop).
    Desktop folders synced via iCloud are accessible but the path is less
    predictable in Shortcuts.

  VERDICT: Go with Option A. Symlinks keep your Mac workflow unchanged.

PART 1: SCRIPTABLE SCRIPT - "Workout Engine"
========================================================================

Install Scriptable from the App Store (free). Create a new script called
"WorkoutEngine". This script does the heavy lifting.

--- BEGIN SCRIPTABLE SCRIPT ---

// WorkoutEngine.js
// Called by Apple Shortcuts with an action parameter

const fm = FileManager.iCloud();
const dir = fm.documentsDirectory(); // This is iCloud Drive/Scriptable/
// We'll use bookmarks or pass file contents from Shortcuts instead.

// When called from Shortcuts, input comes as args.plainTexts or args.shortcutParameter
const input = args.shortcutParameter;

// Parse the input - Shortcuts will pass a JSON string with action + data
let params;
try {
  params = JSON.parse(input);
} catch(e) {
  // If plain text, treat as action name
  params = { action: input };
}

if (params.action === "get_sessions") {
  // Return list of session types for Shortcuts menu
  const programme = JSON.parse(params.programmeJSON);
  const sessions = Object.keys(programme.sessions);
  Script.setShortcutOutput(JSON.stringify(sessions));
}

else if (params.action === "get_exercises") {
  // Return formatted exercise list for display
  const programme = JSON.parse(params.programmeJSON);
  const session = programme.sessions[params.sessionType];

  let display = "";
  display += params.sessionType.toUpperCase() + " DAY\n";
  display += session.muscle_focus + "\n";
  display += "─".repeat(30) + "\n\n";

  session.exercises.forEach((ex, i) => {
    display += (i+1) + ". " + ex.name + "\n";
    display += "   " + ex.sets + " x " + ex.reps;
    if (ex.notes) display += "  (" + ex.notes + ")";
    display += "\n\n";
  });

  Script.setShortcutOutput(display);
}

else if (params.action === "build_prompts") {
  // Return exercise list as JSON for Shortcuts to loop through
  const programme = JSON.parse(params.programmeJSON);
  const session = programme.sessions[params.sessionType];
  Script.setShortcutOutput(JSON.stringify(session.exercises));
}

else if (params.action === "format_csv_row") {
  // Format a single exercise log entry as CSV
  // params: date, sessionType, exerciseName, weight, sets, reps, notes
  const row = [
    params.date,
    params.sessionType,
    params.exerciseName,
    params.weight || "",
    params.sets || "",
    params.reps || "",
    (params.notes || "").replace(/,/g, ";")  // escape commas in notes
  ].join(",");

  Script.setShortcutOutput(row);
}

else if (params.action === "format_all_rows") {
  // Takes the full collected workout data and formats all CSV rows at once
  // params.entries = array of {name, sets, reps, weight, notes}
  // params.date, params.sessionType
  const lines = params.entries.map(e => {
    return [
      params.date,
      params.sessionType,
      e.name,
      e.weight || "",
      e.sets || "",
      e.reps || "",
      (e.notes || "").replace(/,/g, ";")
    ].join(",");
  });

  Script.setShortcutOutput(lines.join("\n"));
}

Script.complete();

--- END SCRIPTABLE SCRIPT ---

NOTE: Save this in Scriptable. It will live in iCloud Drive/Scriptable/WorkoutEngine.js
and sync across devices automatically.

FINAL ARCHITECTURE (PROVEN APPROACH)
========================================================================

COMPONENT 1: Scriptable script "WorkoutEngine"
  - Reads Workout_Program.json from its own iCloud folder
  - Presents all UI (menus, prompts) using Scriptable's Alert/TextField APIs
  - Collects all workout data
  - Returns formatted CSV rows as output to Shortcuts

COMPONENT 2: Apple Shortcut "Log Workout" (very simple)
  - Runs WorkoutEngine via "Run Scriptable Script"
  - Takes the returned CSV text
  - Appends it to Workout_History.csv in iCloud Drive/Shortcuts/
  - Shows confirmation notification

COMPONENT 3: Shortcuts Automation
  - Time-based trigger for the notification/reminder

This is much more reliable because Scriptable handles all the fiddly
string/JSON work natively in JavaScript.


========================================================================
COMPONENT 1: SCRIPTABLE SCRIPT (FINAL VERSION)
========================================================================

Save as "WorkoutEngine" in Scriptable:

--- BEGIN SCRIPT ---

// WorkoutEngine.js - Full workout logging flow
// Reads programme, shows exercises, collects input, returns CSV rows

const fm = FileManager.iCloud();
const baseDir = fm.documentsDirectory();
const programmeFile = fm.joinPath(baseDir, "Workout_Program.json");

// Ensure file is downloaded from iCloud
if (!fm.isFileDownloaded(programmeFile)) {
  await fm.downloadFileFromiCloud(programmeFile);
}

const programmeRaw = fm.readString(programmeFile);
const programme = JSON.parse(programmeRaw);

// STEP 1: Choose session type
const sessionNames = Object.keys(programme.sessions);
const sessionAlert = new Alert();
sessionAlert.title = "Workout Logger";
sessionAlert.message = "Which session are you doing?";
sessionNames.forEach(name => sessionAlert.addAction(name));
sessionAlert.addCancelAction("Cancel");
const sessionIdx = await sessionAlert.presentSheet();

if (sessionIdx === -1) {
  Script.setShortcutOutput("CANCELLED");
  Script.complete();
}

const sessionType = sessionNames[sessionIdx];
const session = programme.sessions[sessionType];

// STEP 2: Show the workout plan
const previewAlert = new Alert();
previewAlert.title = sessionType.toUpperCase() + " - " + session.muscle_focus;
let previewMsg = "";
session.exercises.forEach((ex, i) => {
  previewMsg += (i+1) + ". " + ex.name + " - " + ex.sets + "x" + ex.reps;
  if (ex.notes) previewMsg += " (" + ex.notes + ")";
  previewMsg += "\n";
});
previewAlert.message = previewMsg;
previewAlert.addAction("Start Logging");
previewAlert.addCancelAction("Just Viewing");
const startChoice = await previewAlert.presentAlert();

if (startChoice === -1) {
  Script.setShortcutOutput("VIEWED_ONLY");
  Script.complete();
}

// STEP 3: Loop through exercises and collect data
const now = new Date();
const dateStr = now.getFullYear() + "-"
  + String(now.getMonth()+1).padStart(2,"0") + "-"
  + String(now.getDate()).padStart(2,"0");
const days = ["Sun","Mon","Tue","Wed","Thu","Fri","Sat"];
const dayStr = days[now.getDay()];
const dateCSV = dateStr + " (" + dayStr + ")";

// Also capture start time
const startTime = String(now.getHours()).padStart(2,"0") + ":"
  + String(now.getMinutes()).padStart(2,"0");

const loggedEntries = [];

for (let i = 0; i < session.exercises.length; i++) {
  const ex = session.exercises[i];

  const exAlert = new Alert();
  exAlert.title = ex.name;
  exAlert.message = "Target: " + ex.sets + " x " + ex.reps
    + (ex.notes ? "\nNote: " + ex.notes : "");

  // Different prompts based on exercise type
  if (ex.type === "weight") {
    exAlert.addTextField("Weight (kg)", "");
    exAlert.addTextField("Sets completed", String(ex.sets));
    exAlert.addTextField("Reps achieved", ex.reps);
    exAlert.addTextField("Notes (optional)", "");
  } else if (ex.type === "timed") {
    exAlert.addTextField("Duration achieved", ex.reps);
    exAlert.addTextField("Sets completed", String(ex.sets));
    exAlert.addTextField("Notes (optional)", "");
  } else if (ex.type === "cardio") {
    exAlert.addTextField("Duration", ex.reps);
    exAlert.addTextField("Notes (optional)", "");
  } else {
    // bodyweight
    exAlert.addTextField("Sets completed", String(ex.sets));
    exAlert.addTextField("Reps achieved", ex.reps);
    exAlert.addTextField("Notes (optional)", "");
  }

  exAlert.addAction("Log");
  exAlert.addAction("Skip");
  exAlert.addCancelAction("Done (stop here)");

  const choice = await exAlert.presentAlert();

  if (choice === -1) {
    // User chose to stop early
    break;
  }

  if (choice === 1) {
    // Skip this exercise
    continue;
  }

  // choice === 0: Log it
  let entry = { name: ex.name, weight: "", sets: "", reps: "", notes: "" };

  if (ex.type === "weight") {
    entry.weight = exAlert.textFieldValue(0);
    entry.sets = exAlert.textFieldValue(1);
    entry.reps = exAlert.textFieldValue(2);
    entry.notes = exAlert.textFieldValue(3);
  } else if (ex.type === "timed") {
    entry.reps = exAlert.textFieldValue(0);
    entry.sets = exAlert.textFieldValue(1);
    entry.notes = exAlert.textFieldValue(2);
  } else if (ex.type === "cardio") {
    entry.reps = exAlert.textFieldValue(0);
    entry.notes = exAlert.textFieldValue(1);
  } else {
    entry.sets = exAlert.textFieldValue(0);
    entry.reps = exAlert.textFieldValue(1);
    entry.notes = exAlert.textFieldValue(2);
  }

  loggedEntries.push(entry);
}

// STEP 4: Capture end time
const endNow = new Date();
const endTime = String(endNow.getHours()).padStart(2,"0") + ":"
  + String(endNow.getMinutes()).padStart(2,"0");

// STEP 5: Format as CSV rows
// Header: Date,Session_Type,Exercise,Weight_kg,Sets,Reps,Notes
if (loggedEntries.length === 0) {
  Script.setShortcutOutput("NO_DATA");
  Script.complete();
}

const csvRows = loggedEntries.map(e => {
  // Escape commas and quotes in notes
  let notes = (e.notes || "").replace(/"/g, '""');
  if (notes.includes(",")) notes = '"' + notes + '"';

  return [dateCSV, sessionType, e.name, e.weight, e.sets, e.reps, notes].join(",");
});

// Add a metadata row with timestamps
const metaNote = "Session " + startTime + "-" + endTime;
const metaRow = [dateCSV, sessionType, "SESSION_META", "", "", "", metaNote].join(",");

const output = csvRows.join("\n") + "\n" + metaRow;

// Show summary
const summaryAlert = new Alert();
summaryAlert.title = "Workout Complete";
summaryAlert.message = loggedEntries.length + " exercises logged\n"
  + "Time: " + startTime + " - " + endTime + "\n"
  + "Session: " + sessionType;
summaryAlert.addAction("Save");
summaryAlert.addCancelAction("Discard");
const saveChoice = await summaryAlert.presentAlert();

if (saveChoice === -1) {
  Script.setShortcutOutput("DISCARDED");
  Script.complete();
}

Script.setShortcutOutput(output);
Script.complete();

--- END SCRIPT ---

COMPONENT 2: APPLE SHORTCUT - "Log Workout"
========================================================================

Build this in the Shortcuts app on your iPhone:

ACTION 1: Run Script (Scriptable)
  - Script: WorkoutEngine
  - (Leave parameter empty - the script handles everything)
  -> Output saved as "ScriptResult"

ACTION 2: If [ScriptResult] [is] "CANCELLED"
  -> Stop this shortcut

ACTION 3: If [ScriptResult] [is] "VIEWED_ONLY"
  -> Show Notification: "Workout previewed. Good luck!"
  -> Stop this shortcut

ACTION 4: If [ScriptResult] [is] "NO_DATA"
  -> Show Notification: "No exercises logged."
  -> Stop this shortcut

ACTION 5: If [ScriptResult] [is] "DISCARDED"
  -> Stop this shortcut

ACTION 6: Otherwise (we have CSV data)

ACTION 7: Get File
  - File Path: Shortcuts/Workout_History.csv
  - Service: iCloud Drive
  - If not found, don't error (we'll create it)

ACTION 8: If [File] [has any value]
  -> Get Text from [File]
  -> Set variable "ExistingCSV" to [Text]
ELSE
  -> Set variable "ExistingCSV" to "Date,Session_Type,Exercise,Weight_kg,Sets,Reps,Notes"
END IF

ACTION 9: Text
  -> [ExistingCSV]
     [ScriptResult]
  (This concatenates existing content with new rows, with a newline between)

ACTION 10: Save File
  -> Save [Text] to Shortcuts/Workout_History.csv
  -> Service: iCloud Drive
  -> Overwrite: YES

ACTION 11: Show Notification
  -> Title: "Workout Saved"
  -> Body: "Logged to Workout_History.csv"


========================================================================
COMPONENT 3: AUTOMATION - WORKOUT REMINDER
========================================================================

Apple Shortcuts Automations cannot truly "learn" your schedule. But here's
what's actually possible:

OPTION A: FIXED TIME AUTOMATION (SIMPLEST)
  Go to: Shortcuts app -> Automation tab -> + -> Time of Day
  - Time: e.g., 5:00 PM (or whenever you typically go to the gym)
  - Repeat: Daily
  - Run Immediately: OFF (so it sends a notification you tap to run)
  - When tapped, it runs the "Log Workout" shortcut

OPTION B: DAY-SPECIFIC AUTOMATIONS (BETTER)
  Create separate automations for each training day:

  Monday 5PM    -> Notification: "Push day. Ready to log?"  -> Run "Log Workout"
  Tuesday 5PM   -> Notification: "Pull day. Ready to log?"  -> Run "Log Workout"
  Wednesday 5PM -> Notification: "Leg day. Ready to log?"   -> Run "Log Workout"
  Thursday 5PM  -> Notification: "Push day. Ready to log?"  -> Run "Log Workout"
  Friday 5PM    -> Notification: "Pull day. Ready to log?"  -> Run "Log Workout"
  Saturday 10AM -> Notification: "Cardio + core. Ready?"    -> Run "Log Workout"
  Sunday 10AM   -> Notification: "Active rest / mobility"   -> Run "Log Workout"

OPTION C: LOCATION-BASED (SMARTEST WITHOUT LEARNING)
  Shortcuts -> Automation -> Arrive at [Gym Location]
  - When you arrive at the gym, it notifies you to log your workout
  - Combine with a time condition (e.g., only between 4PM-9PM) to avoid
    false triggers when walking past the gym

ABOUT "LEARNING" YOUR SCHEDULE:
  iOS Focus modes and Siri Suggestions do some pattern learning, but
  Shortcuts can't access that data. True schedule learning would require
  a server-side component or a dedicated app. For now, the day-specific
  automations (Option B) combined with location (Option C) are the most
  practical approach.

CSV FORMAT (MAINTAINED COMPATIBILITY)
========================================================================

The CSV format matches your existing Workout_History.csv:

  Date,Session_Type,Exercise,Weight_kg,Sets,Reps,Notes

Example output from a logged Push session:

  2026-03-25 (Tue),Push,Barbell Bench Press,80,4,6-8,
  2026-03-25 (Tue),Push,Incline DB Press,16,3,10,Felt good
  2026-03-25 (Tue),Push,Seated DB Shoulder Press,12,3,10,
  2026-03-25 (Tue),Push,Lateral Raises,8,3,15,
  2026-03-25 (Tue),Push,Tricep Pushdown,15,3,12,
  2026-03-25 (Tue),Push,Plank,,3,45s,
  2026-03-25 (Tue),Push,Incline Walk,,1,15 min,
  2026-03-25 (Tue),Push,SESSION_META,,,,"Session 17:05-18:15"

The SESSION_META row captures start/end timestamps for training pattern
analysis later. You can filter these out when analysing exercise data.
